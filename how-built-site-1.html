How I build this site - Part 1

Date Fri 13 October 2017 Series Part 1 of How I built this site Tags python / pelican

This is the first part in a multi-part series on how I built this site. In this first post, I'll review the development environment, python, virtual environments, installing the necessary modules and setting up git.

Setting up the development environment

What does that even mean? Before I started building this site I needed to make sure the the necessary software, packages and settings were in place. This starts with setting up a development environment. To me, a development environment is simply what is installed the computer that allows me to get work done. In this case that means the tools necessary to build this site. A list of the tools and technologies are below:

  1. An opperating system: This website was developed using Windows 10, Mac OSX and Linux (Ubuntu 16.04 LTS).
  2. Python: Python 3.5 or 3.6 depending on the system
  3. Anaconda: The Anaconda prompt and Anaconda python distribution
  4. Pelican: A static site generator written in Python
  5. markdown, beautiful soup: Python modules used to support Pelican
  6. pelican-bootstrap3: A theme for Pelican that makes the site look good and play well with phones and tablets
  7. Custom css and javascript: A few custom settings to personalize the pelican-bootstrap3 theme
  8. pelican-plugins: A couple plugins to make tables look nice, allow youtube videos to embed easily and use custom javascript and css in posts
  9. Markdown: the posts on this website are written in markdown, or .md format
  10. Github.com and git: With some many computers used at different times to build the site, a central place where the most up-todate version is kept.
  11. fabric and make: fabric is a python module to automate builds and tasks
  12. Github pages: The server where the static site is hosted

Installing Python (the Anaconda distribution)

I highly recommend installing the Anaconda distribution of python. I have the Anaconda distribution installed on my Windows 10 desktop at work, Macbook Air at home, Ubuntu desktop at home, and my notebook computer at work. Download the lastest release at:

https://www.anaconda.com/download/

Anaconda will have an upto date version of Python (Python 3.6) and the anaconda prompt. The anaconda prompt is usefull because it allows me to run shell commands like I would on my linux machine at home on my Windows 10 machine at work. It makes creating and logging into virtual environmets easy and gives me access to git.

Creating a new virtual enviroment

Before we can start downloading Pelican, we need to create a new virtual environment. Open up the conda prompt or a terminal on Linux or OSX. Type:

$ conda create -n pelican

This will create a new virtual environment called pelican. Becuase I use 4 different computers to work on this site depending on where I am, I need to make sure that the same Python modules are installed on each one. Using virtual environments is one way to ensure that. Once the virtual environment (pelican) is created, we need to activate it and start using it

$ source activate pelican

We should now see (pelican before the terminal prompt.

Install Pelican

Time for some fun! Installing Pelican. Pelican is a python module that creates static websites. Static sites are websites that only have html, css and javascript. A static site is not connected to a database. To get pelican we need to install pip first.

(pelican)$ conda install pip

Once pip is installed, we can install pelican and markdown. These are two of the core peices we need to build the website.

(pelican)$ pip install pelican
(pelican)$ pip install markdown
